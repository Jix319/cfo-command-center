# CFO Command Center - Project Context

## What We Are Building

CFO Command Center is a CFO Operating System.

It is not a generic reporting dashboard and not a UI demo. It is intended to become an operating layer for finance leaders.

The product should help a CFO:

- observe the current state
- analyze what changed
- decide what matters
- act on tasks and exceptions
- report outcomes to leadership

## Primary Users

- CFO
- Financial Controller
- Finance Manager
- Treasury team
- Collections team
- Projects / commercial finance team
- Compliance / statutory team

## Core Experience Goals

The interface should feel:
- enterprise-grade
- information dense without being cluttered
- fast to scan
- stable and predictable
- easy to extend

## Product Philosophy

Every screen should answer at least one of these questions:

- What changed?
- What is at risk?
- What needs action now?
- What is due next?
- What should I report?

If a feature does not support one of those questions, it is probably noise.

## User Journeys

### Observe
The user opens the dashboard to see:
- cash position
- receivables
- collections
- major alerts
- approvals in flight

### Analyze
The user inspects trends:
- sales trend
- cash flow movement
- payment runway
- module-specific charts

### Decide
The user identifies:
- exceptions
- overdue items
- threshold breaches
- large obligations
- blocked approvals

### Act
The user eventually:
- approves
- escalates
- uploads
- filters
- exports
- submits
- reviews

### Report
The user assembles:
- board packs
- status summaries
- monthly close updates
- treasury snapshots
- compliance updates

## Product Design Standards

- Keep the shell stable.
- Keep pages modular.
- Reuse shared primitives.
- Avoid custom one-off visuals unless necessary.
- Prioritize strong hierarchy and fast scanning.
- Preserve dark-mode readability.
- Use data cards, widgets, and charts consistently.

## What This Product Is Not

It is not:
- a consumer app
- a generic admin template
- a toy dashboard
- a design exercise
- a one-page report

## What Success Looks Like

A CFO should be able to open the product and quickly understand:
- where cash stands
- what is late
- what needs approval
- what is upcoming
- what is moving negatively
- what deserves escalation

## Long-Term Direction

The platform should gradually expand into:
- Treasury
- Collections
- Sales
- Projects
- Compliance
- Upload Centre
- AI Assistant
- Bank statement processing
- ERP integrations
- Board packs
